## Session 3: 26-06-2026
### Duration: 70 Minutes
### Tasks:
    - Implemented the Test file for the `HashMap` class to test all the member functions of the `HashMap` class.
### Problem Encountered:
    - The Copy Assignment Operator (operator=) destroys the current object's state before successfully allocating new memory. If the system is under heavy load and new Node*[bucketCount]() throws a std::bad_alloc exception, the assignment operator aborts prematurely.
    - Double-Free: Attempting to free the same memory twice corrupts the heap.

### Solution:
    - Implemented the Copy Assignment Operator (operator=) to first allocate new memory and then free the old memory. This ensures that if the allocation fails, the current object's state remains intact.
    - Implemented a check to ensure that the current object is not the same as the other object before proceeding with the assignment. This prevents self-assignment issues.
